# Projeto - sistema de controle de produtos

## Disciplina: Programação Estruturada

## Grupos: 3 alunos

## Valor: 4 pontos

## Apresentação: ≈10 minutos por grupo

## Data de entrega/apresentação: 17 de novembro

## Descrição:
Cada grupo deverá desenvolver um pequeno sistema de controle de produtos em linguagem C, aplicando os conceitos de vetores, structs, strings, funções e leitura de arquivos texto.

O sistema deve ler os produtos de um arquivo produtos.txt contendo 20 registros e oferecer ao usuário um menu com as seguintes opções:

1. Adicionar produto
2. Buscar produto por código
3. Imprimir produtos
4. Ordenar por preço e imprimir
5. Sair

Cada produto deve possuir código, nome, preço e quantidade. O programa deve ler o arquivo apenas no início da execução, armazenar os dados em memória e não precisa salvar alterações no arquivo.

## Requisitos gerais:
- Uso de funções separadas para leitura, busca, ordenação e impressão.
- Código hospedado no GitHub com README explicando como compilar e executar.

## Apresentação:
Os grupos deverão rodar o programa em sala e explicar o funcionamento. Todos os membros devem demonstrar domínio do código.

## Critérios de avaliação:
- Organização e clareza do código
- Funcionamento
- Explicação e domínio do conteúdo
